# <a name="ediscovery-solutions-in-microsoft-365"></a>eDiscovery-lösningar i Microsoft 365

E-dataidentifiering är processen med att identifiera och leverera elektronisk information som kan användas som bevis i rättsfall. Du kan använda eDiscovery-verktygen i Microsoft 365 för att söka efter innehåll i Exchange Online, OneDrive för företag, SharePoint Online, Microsoft Teams, Microsoft 365-grupper och Yammer-team. Du kan söka i postlådor och webbplatser i samma eDiscovery-sökning och sedan exportera sökresultatet. Du kan använda grundläggande eDiscovery-ärenden för att identifiera, behålla och exportera innehåll som finns i postlådor och webbplatser. Om din organisation har en Office 365 E5- eller Microsoft 365 E5-prenumeration (eller relaterade E5-tilläggsprenumerationer) kan du ytterligare hantera biblioteksarier och analysera innehåll med hjälp av den funktionsrika Advanced eDiscovery-lösningen i Microsoft 365.

## <a name="ediscovery-solutions"></a>eDiscovery-lösningar

Microsoft 365 innehåller tre eDiscovery-lösningar: Innehållssökning, Core eDiscovery och Advanced eDiscovery.

![Viktiga funktioner i Microsoft 365 eDiscovery-verktyg](..\media\m365-ediscovery-solution-graphic.png)

- **Innehållssökning**. Använd verktyget Innehållssökning för att söka efter innehåll Microsoft 365 andra datakällor och exportera sedan sökresultatet till en lokal dator.

- **Core eDiscovery**. Core eDiscovery bygger på de grundläggande sök- och exportfunktionerna i Innehållssökning, vilket gör att du kan skapa eDiscovery-fall och eDiscovery-hanterare som bara kan komma åt de enskilda medlemmarna i ärendet. Med Core eDiscovery kan du koppla sökningar och exporter till ett ärende och låta dig placera ett eDiscovery-värde på innehållsplatser som är relevanta för ärendet.

- **Advanced eDiscovery**. Verktyget Advanced eDiscovery på befintlig ärendehantering, bevarande, sökning och exportfunktioner i Core eDiscovery. Advanced eDiscovery ett end-to-end-arbetsflöde för att identifiera, bevara, samla in, granska, analysera och exportera innehåll som svarar på organisationens interna och externa undersökningar. Det gör det möjligt för juridiska grupper att hantera dokumentelag och arbetsflödet för meddelanden om juridiskt ärende för att kommunicera med dokument dokumenterare som är inblandade i ett ärende. Det gör att du kan samla in och kopiera data från livetjänsten i granskningsuppsättningar, när du kan filtrera, söka och tagga innehåll till icke-relevant innehåll från ytterligare granskning så att arbetsflödet kan identifiera och fokusera på innehåll som är mest relevant. Advanced eDiscovery analys- och maskininlärningsbaserade prediktiv kodningsmodeller för att ytterligare begränsa undersökningens omfattning till det mest relevanta innehållet.

Här är en beskrivning av varje eDiscovery-funktion.

- **Söka efter innehåll**. Sök efter innehåll som lagras i Exchange-postlådor, One Drive för företag-konton, webbplatser SharePoint, Microsoft Teams, Microsoft 365 Grupper och Yammer Teams. Det inkluderar innehåll som genereras av Microsoft 365 appar som lagrar data i postlådor och webbplatser.

- **Nyckelordsfrågor och sökvillkor**. Skapa sökfrågor för KQL-nyckelord om du vill söka efter innehåll som matchar frågevillkor. Du kan också ta med villkor för att begränsa sökningens omfattning.

- **Sökstatistik**. När du har kört en sökning kan du visa statistik för det uppskattade sökresultatet, till exempel antalet och den totala storleken på objekt som matchar dina sökvillkor. Annan statistik omfattar de främsta innehållsplatserna som innehåller sökresultat och antalet objekt som matchar olika delar av sökfrågan.

- **Exportera sökresultat**. Exportera sökresultat till en lokal dator i organisationen i en process i två steg. När du exporterar sökresultat kopieras objekt från den ursprungliga innehållsplatsen i Microsoft 365 till en plats som tillhandahålls Azure Storage Microsoft. Sedan kan du ladda ned objekten till en lokal dator.

- **Rollbaserade behörigheter**. Använd rollbaserad åtkomstbehörighet (RBAC) för att styra vilka eDiscovery-relaterade uppgifter som olika användare kan utföra. Du kan användare till en inbyggd eDiscovery-relaterad rollgrupp eller skapa anpassade rollgrupper som tilldelar specifika eDiscovery-behörigheter.

- **Ärendehantering**. Med e-dataidentifieringsärenden i bas-eDiscovery och Advanced eDiscovery kan du koppla specifika sökningar och exporter till en särskild undersökning. Du kan också tilldela medlemmar till ett ärende för att styra vem som kan komma åt ärendet och visa innehållet i ärendet.

- **Skapa en laglig plats för innehållsplatser**. Bevara innehåll som är relevant för din undersökning genom att sätta ett juridiskt bevarande på innehållsplatserna för ett ärende. På så sätt kan du skydda elektroniskt lagrad information från oavsiktlig (eller avsiktlig) borttagning under din undersökning.

- **Management av insikare**. Hantera de personer som du har identifierat som intressanta personer i ärendet (kallas för biblioteksvårdare) och andra datakällor som inte är associerade med en vårdnadshavare. När du lägger till vårdnadshavare och icke-förfallna datakällor till ett ärende kan du skapa ett juridiskt förvaringsställe för dessa datakällor, kommunicera med de juridiska ombuden genom att använda aviseringsprocessen för juridiska frågor och söka i källinformationskällor och samla in innehåll som är relevant för ärendet.

- **Meddelanden om juridiskt håll**. Hantera processen i vilket man kommunicerar med måls uppringare. En avisering om bevarande av juridiska bestämmelser instruerar biblioteken att bevara innehåll som är relevant för ärendet. Du kan spåra meddelanden som har tagits emot, lästs och bekräftats av bibliotekserare. Med hjälp av kommunikationsarbetsflödet i Advanced eDiscovery kan du skapa och skicka inledande meddelanden, påminnelser och eskalering om det inte går att bekräfta en avisering om väntande samtal.

- **Avancerad indexering**. När du lägger till namnbaserade och icke-systemrelaterade datakällor indexeras de associerade innehållsplatserna om (i en process som kallas Avancerad *indexering)* så att allt innehåll som anses vara delvis indexerat bearbetas på nytt, så att det blir fullt sökbart när du samlar in data för undersökning.

- **Felreparation**. Åtgärda bearbetningsfel med hjälp av en process *som kallas felreparation.* Med felreparation kan du hantera dataproblem som hindrar Advanced eDiscovery från att bearbeta innehållet på rätt sätt under Avancerad indexering. Till exempel kan inte lösenordsskyddade filer bearbetas eftersom filerna är låsta eller krypterade. Med hjälp av felreparation kan du ladda ned filer med fel, ta bort lösenordsskyddet och sedan ladda upp de åtgärdade filerna.

- **Granska uppsättningar**. Lägg till relevanta data i en granskningsuppsättning. En granskningsuppsättning är en säker, Microsoft-Azure Storage plats i Microsoft-molnet. När du lägger till data i en granskningsuppsättning kopieras de insamlade objekten från deras ursprungliga innehållsplats till granskningsuppsättningen. Med granskningsuppsättningar får du en statisk, känd uppsättning innehåll som du kan söka efter, filtrera, tagga, analysera och förutsäga relevans med prediktiv kodningsmodeller. Du kan också spåra och rapportera vilket innehåll som läggs till i granskningsuppsättningen.

- **Stöd för molnbilagor och SharePoint versioner.** När du lägger till innehåll i en granskningsuppsättning kan du inkludera molnbilagor eller länkade filer. Det innebär att målfilen för en molnbilaga eller länkad fil läggs till i granskningsuppsättningen. Du kan också välja att lägga till alla versioner av SharePoint dokument i en granskningsuppsättning.

- **Optisk teckenläsning (OCR)**. När innehåll läggs till i en granskningsuppsättning extraherar OCR-funktionerna text från bilder och inkluderar bildtexten med det innehåll som har lagts till i en granskningsuppsättning. Då kan du söka efter bildtext när du ställer frågor mot innehållet i granskningsuppsättningen.

- **Konversationstrådning**. När chattmeddelanden från Teams och Yammer konversationer läggs till i en granskningsuppsättning kan du välja att samla in hela konversationstråden. Det innebär att hela chattkonversationen som innehåller objekt som uppfyller samlingsvillkoren läggs till i granskningsuppsättningen. På så sätt kan du granska chattobjekt i samband med konversationen fram och tillbaka.

- **Insamlingsstatistik och rapporter**. När du har skapat en utkastsamling eller skapat en kom ihåg en samling i en granskningsuppsättning kan du visa en omfattande uppsättning statistik för hämtade objekt, till exempel innehållsplatser som innehåller flest objekt som matchade sökvillkoren och antalet objekt som returnerades av sökfrågan. Du kan också förhandsgranska en delmängd av resultaten. Det här inkluderar även antalet underordnade objekt som har extraherats från sina överordnade objekt och lagts till som separata objekt i granskningsuppsättningen.

- **Granska ställa in filtrering**. När innehållet har lagts till i en granskningsuppsättning kan du använda filter för att endast visa den uppsättning objekt som matchar filtreringskriterierna. Sedan kan du spara filteruppsättningarna som en fråga, vilket gör att du snabbt kan återanvända de sparade filtren. Granska ställa in filtrering och sparade frågor för att snabbt ta fram innehåll till de objekt som är mest relevanta för din undersökning.

- **Tagga**. Taggar hjälper dig även att identifiera innehåll som inte är relevant och att identifiera det mest relevanta innehållet. När experter, jurister eller andra användare granskar innehåll i en granskningsuppsättning kan deras åsikter om innehållet fångas upp med hjälp av taggar. Om syftet till exempel är att undvika onödigt innehåll kan en användare tagga dokument med en tagg, till exempel "svarar inte". När innehåll har granskats och taggats kan du skapa en granskningsuppsättningsfråga för att utesluta innehåll som är taggat som "icke-responsivt". Med den här processen slipper du det icke-responsiva innehållet från efterföljande steg i eDiscovery-arbetsflödet.

- **Analys**. Advanced eDiscovery innehåller verktyg för att analysera dokument med granskningsuppsättning så att du kan organisera dokumenten på ett sammanhängande sätt och minska mängden dokument som ska granskas. *Med en text som liknar* varandra i närheten av dubblettidentifieringsgrupper blir granskningsprocessen effektivare. *E-posttråd* identifierar specifika e-postmeddelanden som ger en fullständig kontext för konversationen i en e-posttråd. *Med* funktionen Teman försöker du analysera teman som diskuteras i granska ange dokument och tilldela ett tema till dokument så att du kan granska dokument som har ett relaterat tema. Med hjälp av analysfunktionerna blir granskningsprocessen effektivare så att granskare kan granska endast en del insamlade dokument.

- **Prediktiv kodningsmodeller**. Använd prediktiv kodningsmodeller för att minska och minska stora volymer av ärendeinnehåll till en relevant uppsättning objekt som du kan prioritera för granskning. Detta sker genom att skapa och träna egna prediktiv kodningsmodeller som hjälper dig att prioritera granskningen av de mest relevanta objekten i en granskningsuppsättning. Utbildningen används för att använda förutsägelseresultat för alla objekt i granskningsuppsättningen. På så sätt kan du filtrera objekt baserat på förutsägelseresultatet, så att du först kan granska de mest relevanta (eller icke-relevanta) elementen.

- **Beräknade metadata för dokument**. Många Advanced eDiscovery till metadataegenskaper, till exempel avancerad indexering, konversationstrådning, analys och förutsägelsekodning, lägger till metadataegenskaper för att granska uppsättningsdokument. Dessa metadata innehåller information som är relaterad till funktionen som utförts av en specifik funktion. När du granskar dokument kan du filtrera på metadataegenskaper för att visa dokument som matchar dina filtervillkor. Dessa metadata kan importeras till granskningsprogram från tredje part när vissa dokument har exporterats.

- **Genomskinlighet för långvariga jobb.** Jobb i Advanced eDiscovery är, vanligtvis långvariga processer som utlöses av användaråtgärder, till exempel att lägga till granskare till ett ärende, lägga till innehåll i en granskningsuppsättning, köra analys och förutsägande kodningsmodeller för utbildning. Du kan spåra status för de här jobben och få supportinformation om du behöver eskalera problem till MS-supporten.

- **Exportera till kundägd Azure Storage plats.** När du exporterar dokument från en granskningsuppsättning kan du exportera dem till ett Azure Storage som hanteras av din organisation. Dessutom kan Advanced eDiscovery anpassa vilka data som exporteras. Det omfattar export av filmetadata, interna filer, textfiler, taggar och omformaterade dokument som har sparats i en PDF-fil.

## <a name="ediscovery-subscription-comparison"></a>jämförelse av eDiscovery-prenumerationer

I följande avsnitt visas minimikraven för prenumeration för innehållssökning, bas-eDiscovery och Advanced eDiscovery. Prenumerationer som har stöd för bas-eDiscovery har också stöd för innehållssökning. Prenumerationer som har Advanced eDiscovery även stöd för innehållssökning och bas-eDiscovery.

### <a name="content-search"></a>Innehållssökning

- Microsoft 365 E1-prenumeration
- Microsoft 365 G1-prenumeration
- Office 365 Education A1-abonnemang

### <a name="core-ediscovery"></a>Core eDiscovery

- Microsoft 365 E3 prenumeration
- Microsoft 365 G3 prenumeration
- Microsoft 365 Education A3- eller Office 365 Education A3-prenumeration

### <a name="advanced-ediscovery"></a>Advanced eDiscovery

- Microsoft 365 E5 eller Office 365 E5-prenumeration
- Microsoft 365 E3-abonnemang med E5-tillägget för efterlevnad
- Microsoft 365 E3-prenumeration med E5-tillägget eDiscovery och granskning
- Microsoft 365 G5-prenumeration
- Microsoft 365 G5-prenumeration med G5-efterlevnads tillägg
- Microsoft 365 G5-prenumeration med G5-tillägget eDiscovery och granskning
- Microsoft 365 Education A5 eller Office 365 Education A5-prenumeration
