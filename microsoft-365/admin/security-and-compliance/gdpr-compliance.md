# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>GDPR förenklat: En guide för ditt småföretag

 *Använda Microsoft 365 för företag för att minska och hantera GDPR-efterlevnad* 
  
Den allmänna dataskyddsförordningen (GDPR) är en EU-förordning som föreskriver hur en organisation ska hantera personuppgifter. GDPR kommer att träda i kraft den 25 maj 2018. Om ditt företag säljer till, tillhandahåller tjänster till eller anställer eu-medborgare kommer [GDPR](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en) att påverka dig. 
  
Den här artikeln hjälper dig att förstå vad GDPR är, varför det kom till och hur Microsoft 365 för företag kan hjälpa din organisation att följa GDPR.
  
## <a name="a-quick-overview-of-the-gdpr"></a>En snabb översikt över GDPR

GDPR är en EU-förordning som uppdaterar och utvidgar det tidigare dataskyddsdirektivet (DPD) som först antogs 1995. GDPR handlar om sekretessen för en individs data, vara den enskilde en kund, kund, anställd eller affärspartner. GDPR:s mål är att stärka skyddet av personuppgifter för EU-medborgare, oavsett om de är bosatta i EU eller någon annanstans. I förordningen fastställs förväntningar och råd om hur de ska uppnås. När GDPR blir verkställbart i slutet av maj 2018 måste organisationer ha åtgärder som uppfyller kraven i GDPR.
  
GDPR handlar om data och hur den används. Tänk på data som har en livscykel. Cykeln startar när du samlar in data, fortsätter när du lagrar den och använder den (bearbetning) och slutar när du helt tar bort den från dina system. 
  
GDPR handlar om följande typer av data: 
  
- **Personuppgifter:** Om du kan länka data till en individ och identifiera dem, anses dessa uppgifter vara personliga med avseende på GDPR. Exempel på personuppgifter är namn, adress, födelsedatum och IP-adress. GDPR anser även kodad information (även känd som "pseudonyma" information) vara personuppgifter, oavsett hur dunkla eller tekniska uppgifterna är, om uppgifterna kan kopplas till en individ.
    
- **Känsliga personuppgifter** Detta är data som lägger till mer information till personuppgifter. Exempel är religion, medlemskap i fackförening, etniskt ursprung och så vidare. Känsliga personuppgifter omfattar även biometriska uppgifter och DNA. Enligt GDPR har känsliga data strängare skyddsregler än personuppgifter. 
    
## <a name="gdpr-terms"></a>GDPR-termer

Du ser några termer som ofta refereras till i GDPR. Det är viktigt att förstå dessa termer.
  
 **Medgivande**
  
GDPR säger: "Behandlingen av personuppgifter bör utformas för att tjäna mänskligheten." GDPR hoppas kunna uppnå detta mål genom att använda samtycke vid behandling av personuppgifter. Det kan vara den enkla handlingen att fråga dina kunder om de vill ta emot e-postmeddelanden från ditt företag. Det innebär också att inga fler opt-out kryssrutor på din webbplats när du vill använda data för marknadsföring. Du måste ta uttryckligt samtycke med hjälp av en "tydlig jakande handling". Och du måste också föra register över när ett samtycke tas eller återkallas.
  
 **Registrerades rättigheter**
  
GDPR fastställer registrerade rättigheter, vilket innebär att kunder, anställda, affärspartners, kunder, entreprenörer, entreprenörer, studenter, leverantörer och så vidare har rätt att:
  
- **Informeras om deras uppgifter:** Du måste informera enskilda personer om din användning av deras data. 
    
- **Ha tillgång till deras data:** Du måste ge enskilda personer tillgång till någon av deras data som du har (till exempel genom att använda kontoåtkomst eller på något manuellt sätt). 
    
- **Be om datakorrigering:** Enskilda personer kan be dig att korrigera felaktiga uppgifter. 
    
- **Be om att data tas bort:** Även känd som "rätten till radering", tillåter denna rätt en individ att begära att någon av deras personuppgifter ett företag har samlat in raderas över alla system som använder den eller dela den. 
    
- **Begär begränsad behandling:** En person kan be dig att undertrycka eller begränsa deras data. Den är dock endast tillämplig under vissa omständigheter. 
    
- **Har dataportabilitet:** En person kan begära att deras uppgifter överförs till ett annat företag. 
    
- **Objekt:** En individ kan invända mot att deras data används för olika användningsområden, inklusive direktmarknadsföring. 
    
- **Be att inte bli föremål för automatiserat beslutsfattande, inklusive profilering:** GDPR har strikta regler om hur du använder data för att profilera personer och automatisera beslut baserat på den profileringen. 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>Vanliga småföretagsfrågor om GDPR

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>Jag är en enskild ägare - måste jag verkligen oroa mig för GDPR?

GDPR handlar om de data du behandlar, inte antalet anställda du har. Det påverkar företag av alla storlekar, även enmansföretag. Företag med färre än 250 anställda har dock vissa undantag, till exempel minskad journalföring, men bara om du är säker på att databehandlingen inte påverkar individens rättigheter och är tillfällig behandling.
  
Till exempel skulle behandling av icke-personuppgifter vara undantagen eller behöva reducerade åtgärder. Men om du bearbetar data som ses som "känsliga data för särskild kategori", även om det bara ibland, måste du registrera den här databehandlingen. Definitionen av tillfällig bearbetning är, men det är tänkt att gälla för data som används en eller sällan.
  
Du bör också se till att personuppgifter som du samlar in skyddas. Det innebär att du måste kryptera den och se till att åtkomsten till den kontrolleras med minst ett lösenord. Att hålla dina kunddata på ett kalkylblad på skrivbordet utan skydd uppfyller inte GDPR:s förväntningar. 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>Hur vet jag om vårt företags webbplats är GDPR-kompatibel?

Den första frågan att ställa dig är: Samlar du in personuppgifter någonstans på din webbplats? Du kan till exempel ha ett kontaktformulär som frågar efter ett namn och en e-postadress. Om du vill skicka marknadsföringsmeddelanden kontrollerar du att du lägger till en kryssruta för opt-in som förklarar exakt vad du ska använda data till. Endast om mottagaren kontrollerar den rutan kan du använda deras personuppgifter i marknadsföringssyfte.
  
Kontrollera också att databasen som lagrar data är skyddad. Ditt webbhotell eller moln lagring leverantör kommer att kunna ge råd om detta. Om du använder Microsoft 365 för företag är lagring av data GDPR-kompatibel. 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>Mitt företag är utanför Europa. Påverkar gdpr verkligen oss?

GDPR är en förordning som skyddar EU-medborgarna. Om ditt företag handlar med EU-medborgare nu, eller om du hoppas på i framtiden, kommer du att påverkas. Detta gäller både medborgare som bor i en EU-stat och de som bor någon annanstans.
  
Tänk på följande exempel:
  
- Ett amerikanskt företag som hyr bilar till EU-medborgare måste uppfylla GDPR-kraven när de samlar in och behandlar kundens data. Företaget kommer att vara skyldigt att ta samtycke när de tar kundens data och se till att uppgifterna lagras säkert. De måste också se till att kunden kan tillämpa alla sina registrerade rättigheter. 
    
- Ett australiskt företag säljer produkter online, och dess användare inrätta online-konton. GDPR-registrerades rättigheter och samtycke kommer att tillämpas på EU-medborgare som öppnar ett konto. Företaget måste se till att kunden kan tillämpa alla sina registrerade rättigheter. 
    
- En internationell välgörenhetsorganisation samlar in data om givare och använder dem för att skicka ut uppdateringar och förfrågningar om donationer. GDPR säger: "... Behandling av personuppgifter för direkt marknadsföring kan anses utföras av ett berättigat intresse." Men ansvaret ligger på organisationen att bevisa sina intressen åsidosätta den registrerades intressen. Företaget (eller i detta fall välgörenhetsorganisationen) bör alltid få informerat, uttryckligt, opt-in samtycke.
    
GDPR gäller även om kunddata flyttas över gränserna. Om du använder cloud computing för datalagring måste du se till att tjänsten är helt GDPR-kompatibel. Det kan bli komplicerat om datalagring finns på platser som har en dålig historia av dataskydd. Om du använder Microsoft 365 för företag har vi rätt juridisk dokumentation för att täcka GDPR-krav.
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>Visst, jag samlar in data, men något annat företag lagrar den. Får det mig att komma undan?

Enligt GDPR, om du samlar in data påverkas du i viss utsträckning. GDPR har begreppet personuppgiftsbiträde och personuppgiftsansvarig: 
  
- **Personuppgiftsansvarig:** En individ eller organisation (du kan ha gemensamma registeransvariga) som bestämmer hur, vad och varför data samlas in. De kan lagra den med ett annat företags molnservrar. Till exempel är en webbplats som samlar in kunddata en controller. 
    
- **Databehandlare:** En person eller organisation som lagrar data för den eller de registeransvariges räkning och behandlar dessa data på begäran. Microsoft 365 Apps for business-datalagring fungerar till exempel som processor och är helt GDPR-kompatibel. 
    
    En organisation eller ett system kan fungera både som en styrenhet och en processor. Microsoft 365 för företag kan fungera som både och följer GDPR.
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>Kan jag fortfarande skicka ut marknadsföringsmeddelanden till mina gamla kunder?

Du måste se till att dina kunder, även de som du har haft i flera år, har samtyckt till att använda sina data för marknadsföring. Du kan tidigare ha fångat samtycke, samt en post för att visa det. Om så är fallet är du redo att fortsätta marknadsföra. Om inte, måste du få tillstånd från kunden att fortsätta marknadsföra till dem. Detta innebär vanligtvis att skicka ett e-postmeddelande ber kunderna att gå till din webbplats och välja ett alternativ för att samtycka till att ta emot framtida e-postmeddelanden. 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>Måste jag oroa mig för GDPR när jag rekryterar nya medarbetare? Hur är det med nuvarande anställda?

GDPR påverkar inte bara kunddata. det sträcker sig till medarbetardata också. Nya rekryter finns ofta med hjälp av sociala medier som LinkedIn. Se till att du inte lagrar några potentiella rekryteringsdata utan deras uttryckliga tillstånd.
  
När det gäller befintliga anställda och nya anställningsavtal förutsätter en underskrift vid avtalets inte nödvändigtvis samtycke, särskilt inte när en icke-positiv klausul används i ett avtal. I det här fallet måste du samla in samtycke på ett uttryckligt sätt som är associerat med klausulen. Vad detta innebär beror på ditt anställningsavtal, men du kan använda "berättigat intresse" i vissa fall och lägga till en anställd databehandling meddelande för att se till att dina anställda är medvetna om vad du ska göra med sina data.
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-for-business"></a>Tillgodose sekretessproblem med Microsoft 365 för företag

Att bli kompatibel med GDPR handlar om att se till att personuppgifter skyddas. GDPR har ett koncept som kallas Privacy by Design och Default. Detta innebär att dataskyddet bör "bakas in" i ett system och en produkt så att det är andra natur att tillgodose integritetsfrågor. 
  
Liksom sina större motsvarigheter behöver ett litet företag bekvämlighet utan att offra säkerheten. Microsoft 365 för företag är utformat för företag med färre än 300 anställda. Små företag kan använda Microsofts molnbaserade verktyg för att förbättra företagets produktivitet. Med Microsoft 365 för företag kan ett litet företag hantera e-postmeddelanden, dokumentation och till och med möten och evenemang. Den har också inbyggda säkerhetsåtgärder och enhetshantering, som är avgörande för GDPR-efterlevnad.
  
Microsoft 365 för företag kan hjälpa dig med GDPR-processen på följande sätt:
  
- **Upptäck:** Ett viktigt steg till GDPR-efterlevnad är att veta vilka data du har. 
    
- **Hantera:** Att kontrollera åtkomsten till data och hantera dess användning är en integrerad del av GDPR. Microsoft 365 för företag skyddar affärsdata baserat på principer som du vill tillämpa på enheter. Enhetshantering är viktigt i en tid där anställda arbetar på distans. Microsoft 365 för företag innehåller enhetshanteringsfunktioner som säkerställer att data är skyddade på alla enheter. Du kan till exempel ange att alla Windows 10-enheter i ditt företag är skyddade via Windows Defender. 
    
- **Skydda:** Microsoft 365 för företag är utformat för säkerhet. Dess enhetshanterings- och dataskyddskontroller fungerar i hela företagsnätverket, inklusive fjärrenheter, för att skydda data. Microsoft 365 för företag erbjuder kontroller som sekretessinställningar i Office-program och kryptering av dokument. Med Microsoft 365 för företag kan du utföra GDPR-efterlevnadsövervakning för att se till att du har rätt skyddsnivå inställd. 
    
- **Rapport:** GDPR lägger stor vikt vid rapportering. Även ett företag med en enda anställd, om den verksamheten bearbetar stora mängder data, krävs för att dokumentera och rapportera om sina procedurer. Microsoft 365 för företag tar huvudvärk av rapporteringskrav för mindre organisationer. 
    
    Med verktyg som granskningsloggar kan du spåra och rapportera om datarörelser. Rapporterna omfattar klassificering av data som du samlar in och lagrar, vad du gör med data och överföringar av data. 
    
Kunder, anställda och kunder blir mer medvetna om vikten av datasekretess och förväntar sig nu att ett företag eller en organisation respekterar den integriteten. Microsoft 365 för företag ger dig verktyg för att uppnå och upprätthålla GDPR-efterlevnad utan en massiv omvälvning för ditt företag.
