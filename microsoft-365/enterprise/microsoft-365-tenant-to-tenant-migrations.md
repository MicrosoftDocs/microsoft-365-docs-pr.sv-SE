# <a name="microsoft-365-tenant-to-tenant-migrations"></a>Microsoft 365-migrering till klient organisation

Det finns flera arkitektur funktioner för fusioner, förvärv, divestitures och andra scenarier som kan leda till att du migrerar en befintlig Microsoft 365-klient organisation till en ny klient organisation. De flesta kunder kan samar beta med Microsoft Consulting Services eller en Microsoft-partner för att migrera klient organisationer, inklusive användning av verktyg från tredje part för att migrera innehåll. 

Använd en [arkitektur modell för klient](../downloads/Microsoft-365-tenant-to-tenant-migration.pdf) organisations klient organisation för att förstå hur du planerar för migrering av Microsoft 365-klient organisationer och stegen i en migrering.

[![Användarmigrering för klient organisation](../media/solutions-architecture-center/msft-tenant-to-tenant-migration-thumb.png)](../downloads/Microsoft-365-tenant-to-tenant-migration.pdf) 

Du kan också ladda ned den här modellen i [PDF-](https://github.com/MicrosoftDocs/microsoft-365-docs/raw/public/microsoft-365/downloads/Microsoft-365-tenant-to-tenant-migration.pdf) eller [Visio](https://github.com/MicrosoftDocs/microsoft-365-docs/raw/public/microsoft-365/downloads/Microsoft-365-tenant-to-tenant-migration.vsdx) -format och skriva ut den på Letter, Legal eller Tabloid (11 x 17) pappers storlek.

Den här modellen ger vägledning och en utgångs punkt för planering med avsnitt på:

- Mappning av affärs scenarier till arkitektur metoder
- Utformning

Den här modellen innehåller också detaljerade exempel på:

- Ett migrerings flöde
- Ett steg i fasen migration
- Ett klient organisations-eller flödes schema
